## 3. 진입 화면

### [SPLASH-01] 스플래시
- 1초 노출, 로고 + 로딩 인디케이터
- 자동 로그인 토큰 검증
- 검증 OK → 메인 탭
- 검증 실패 또는 신규 → 온보딩

### [ONBOARD-01] 온보딩
- 3장 슬라이드 (스와이프 또는 다음 버튼)
- 스킵 가능 → 메인 탭 (비로그인)
- 마지막 슬라이드 "시작하기" → Apple Sign In

### [SIGNUP-01] 초기 설정 (가입 직후)
가입 직후 3단계로 진행:

**Step 1: 프로필 설정**
- 닉네임 (2~12자)
- 프로필 사진 (선택)

**Step 2: 모드 선택**
- 모델 모드: 예시 이미지로 디자인 미리보기
- 실착 모드: 실제 시술 사진으로 보기
- ※ 프로필에서 언제든 변경 가능

**Step 3: 관심 태그**
- 표준 태그 사전에서 최소 3개 ~ 최대 8개 선택
- 카테고리별 (스타일/색상/분위기/계절)
- 스킵 가능 (나중에 프로필에서 설정)

## 4. 메인 화면별 명세

### 🏠 [HOME-01] 홈
**진입:** 앱 실행 후 기본 화면, 탭바 홈 클릭

**구조:**
- [헤더] 로고 + 🔔 + ♡
- [검색바] 🔍 어떤 디자인을 찾으세요?
- [섹션 1: 카테고리] 가로 스크롤 칩
- [섹션 2: 이번 주 추천] 카드 가로 스크롤 + 더 보기
- [섹션 3: 인기 네일샵] 샵카드 가로 스크롤 + 더 보기
- [섹션 4: 내 취향 추천] 로그인만 / 디자인 그리드 2열
- [섹션 5: 인기 스네일] 선택 / 가로 스크롤

**상태 변형:**
- 비로그인: 섹션 4 숨김 + 하단에 "로그인하고 맞춤 추천 받기" 배너
- 로딩: 섹션별 스켈레톤 UI
- 데이터 0: 카테고리 칩만 노출

### 🔍 [SEARCH-01] 탐색 (검색)
**진입:** 홈 검색바 탭, 카테고리 칩 탭

**구조:**
- [헤더] ← + 검색바(자동 포커스) + 취소
- [탭] 디자인 / 샵 / 리뷰
- [필터 칩 가로 스크롤] 지역/가격/소요시간/색상/분위기/날짜
- [정렬] 인기순 ▼
- [결과] 활성 탭에 따라 (디자인: 그리드 2열 / 샵: 리스트 / 리뷰: 카드 리스트) + 무한 스크롤

**검색바 상태:**
- 입력 전: 최근 검색어 + 인기 검색어
- 입력 중: 자동완성 (태그 사전 기반)
- 입력 후: 결과 (3개 탭으로 분리)

**상태 변형:**
- 검색 결과 0건: "유사한 디자인을 추천해드려요" + 유사 디자인 그리드
- 로딩: 스켈레톤
- 에러: "다시 시도" 버튼

### 🎨 [DESIGN-01] 디자인 상세 ⭐ (AI 태그 노출 X)
**진입:** 디자인 카드 탭

**구조:**
- [헤더] ← + 공유 + ♡
- [이미지 갤러리] 큰 이미지 스와이프 + 페이지 인디케이터
- [기본 정보] 디자인명 / 가격 / 소요시간
- [샵 정보 카드] 샵 썸네일 + 샵명 + 평점 + 지역 + 샵 보기
- [섹션1: 비슷한 디자인] 그리드 2열
- [섹션2: 이 디자인을 받은 스네일] 가로 스크롤 + 전체 보기
- [섹션3: 이 디자인 리뷰] 평점 + 리뷰 카드 1~3개 + 전체 리뷰 보기
- [하단 고정 CTA] ♡ / 공유 / 예약하기

**상태 변형:**
- 비로그인: 찜/예약 탭 시 로그인 모달
- 시술 가능 디자이너 0명: "현재 예약 불가" 배지, 예약 버튼 비활성
- 사장님이 태그 안 단 경우: 태그 섹션 숨김
- **AI 태그 절대 노출 X** (사장님이 직접 단 태그만)

### 📅 [BOOKING-01] 예약 화면
**진입:** 디자인 상세 → 예약하기

**구조:**
- [헤더] ← 예약하기
- [선택한 디자인 요약]
- [Step 1: 날짜 선택] 캘린더 (영업일만 활성, 휴무일 회색)
- [Step 2: 시간 선택] 날짜 선택 후 노출, 만석 표시
- [Step 3: 디자이너 선택] 자동배정 or 디자이너 선택 (가능한 디자이너만)
- [요청사항] 선택, 최대 200자
- [취소/노쇼 안내] 필수 확인 체크박스
- [하단 고정 CTA] 예약 요청 보내기

**완료 후:**
- 자동수락 샵: "예약이 확정되었어요" 토스트 → 일정 탭
- 수동수락 샵: "사장님 승인 대기 중" 토스트 → 일정 탭

**상태 변형:**
| 케이스 | 표시 |
|---|---|
| 활성 예약 3건 이상 | 모달: "예약 한도 초과" |
| 노쇼 2회 누적 | 배너 경고 + 진행 가능 |
| 노쇼 3회 누적 | 모달: "예약이 제한되었어요" + 진행 불가 |
| 같은 샵/날짜/시간대 중복 | 모달: "이미 비슷한 시간에 예약이 있어요" |
| 가능 디자이너 0명 | 시간 슬롯 만석 표시 |

### 📅 [SCHEDULE-01] 일정 탭
**진입:** 탭바 일정 (비로그인 시 로그인 모달)

**구조:**
- [헤더] 내 예약 + 🔔
- [상단 토글] 예정 / 지난 예약

**카드 상태별 표시:**
| 상태 | 표시 | 색상 | 액션 |
|---|---|---|---|
| pending | "승인 대기 중" | 노란색 | 취소만 |
| confirmed | "✓ 확정" | 초록색 | 상세/취소 |
| rejected | "거절됨" | 회색 | 상세(사유 확인) |
| cancelled_by_user | "취소됨" | 회색 | 상세 |
| cancelled_by_shop | "샵 사정 취소" | 회색 | 상세 |
| completed | "완료" + 리뷰 CTA | 파란색 | 리뷰 작성/상세 |
| no_show | "노쇼 처리" | 빨강 | 상세 |

**지난 예약 탭:**
- D+Day 표시
- 리뷰 안 쓴 completed는 상단 고정 + 리뷰 작성 강조
- 하단 "다음 네일 예상일: D+21" 안내

### 📋 [SCHEDULE-02] 예약 상세
**진입:** 일정 탭의 예약 카드 탭

**구조:**
- [헤더] ← 예약 상세
- [상태 배지] + D-Day
- [디자인 정보] 썸네일 + 디자인명 + 가격 + 소요시간 + 디자인 보기
- [샵 정보] 썸네일 + 주소 + 전화 + 길찾기/전화/샵 보기
- [예약 정보] 날짜 / 시간 / 디자이너
- [요청사항]
- [거절/취소 사유] 해당 시

**하단 CTA (상태별):**
- pending/confirmed: 예약 취소
- completed: 리뷰 작성하기
- 기타: 버튼 없음

### ⭐ [REVIEW-01] 리뷰 작성
**진입:** 일정 탭 / 예약 상세 → 리뷰 작성

**구조:**
- [헤더] ← 리뷰 작성
- [예약 정보 요약]
- [별점] 필수, ☆☆☆☆☆ / 디자이너 별점 없음, 샵 별점만
- [본문] 필수, 최소 10자 최대 1,000자
- [사진] 선택, 0~5장
- [태그] 선택, 최대 5개 (표준 태그에서 선택)
- [등록하기]

**제약:**
- completed 상태 예약만 작성 가능
- 1예약 = 1리뷰
- 작성 후 7일 내 수정/삭제 가능

### ❄️ [SNAP-01] 스네일 탭 (커뮤니티)
**진입:** 탭바 스네일

**구조:**
- [헤더] 스네일 + 🔔 + ✏️
- [상단 토글] 스네일 / 랭킹 / 팔로잉
- [스네일 탭] 세부 토글: 최신 / 추천

**스네일 카드:**
- 작성자 정보 (사진/닉네임)
- 인증 뱃지 (예약 연결 시)
- 이미지 (정사각형)
- 캡션 (2줄, 더 보기)
- 태그
- ♡ 좋아요 / 💬 댓글 / 👁 조회수
- "이 디자인 받기" CTA (디자인 태그된 경우)
- 무한 스크롤

**탭별 차이:**
- 스네일: 최신/추천 (popularity_score 기반)
- 랭킹: 좋아요 많은 순 (일/주/월 단위)
- 팔로잉: 팔로우한 유저의 스네일만 (0명일 때 추천 유저 노출)

**상태 변형:**
- 비로그인: 좋아요/댓글/팔로우 시 로그인 모달
- 데이터 0: "첫 스네일을 올려보세요" CTA

## 5. 공통 컴포넌트 정의

### 5.1 디자인 카드 (Design Card)
- 썸네일 정사각형
- 우측 상단 찜 ♡
- 네일샵명 · 지역
- 가격 · 소요시간

### 5.2 샵 카드 (Shop Card)
- 대표 이미지
- 샵명
- 평점 · 지역
- 디자인 N개

### 5.3 스네일 카드 (Snap Card)
- 프사 + 닉네임 + 팔로우
- 인증 뱃지 (해당 시)
- 이미지 정사각형
- 캡션 2줄
- 태그
- ♡ 좋아요 / 💬 댓글

### 5.4 예약 카드 (Reservation Card)
- D-Day (상태별 색상)
- 디자인 썸네일
- 샵명
- 날짜 시간
- 상태 배지
- 상세 / 취소 버튼

### 5.5 리뷰 카드 (Review Card)
- 프사 + 닉네임
- 별점 · 날짜
- 본문
- 사진 썸네일
- 태그
- 샵 답변 (있을 시)

### 5.6 빈 상태 (Empty State)
- 일러스트 / 아이콘
- 안내 텍스트
- CTA 버튼

## 6. 모달 / 시트

### 6.1 로그인 모달
- 트리거: 비로그인 사용자가 찜/예약/스네일 작성 등 시도
- Apple Sign In 버튼 + "나중에"

### 6.2 필터 시트
- 트리거: 탐색 화면 필터 칩
- 하단 바텀 시트 (50% 높이)

### 6.3 신고 모달
- 신고 사유 6개: 스팸/광고, 부적절/혐오, 허위정보, 사칭, 저작권 침해, 기타
- 상세 사유 입력 (선택, 최대 500자)

### 6.4 공유 시트
- 시스템 네이티브 공유 시트

### 6.5 모드 변경 시트
- 모델 모드 / 실착 모드 토글

### 6.6 예약 한도/노쇼 경고 모달
- 케이스별 다른 문구 (BOOKING 섹션 참조)

### 6.7 폴더 만들기/이동 시트
- 폴더 이름 입력 + 커버 이미지 선택

## 10. 개발 주의사항

- **AI 태그 노출 X**: 디자인 상세에 AI가 매긴 태그/색상/분위기/스타일 노출하지 않음. 사장님이 직접 단 태그만 노출 (있는 경우만)
- **컴포넌트 정의**: 5번 섹션의 카드를 먼저 만들면 그리드/리스트 빠르게 구현 가능
- **상태 변형**: 메인 화면 다 만들고 나서 상태별로 따로 처리
- **공통 헤더/탭바**: 한 번 만들어서 컴포넌트화
- **예약 카드 7가지 상태**: 색상 + 배지 다르게 처리
